# 02-信息编号与置信度规则

## 编号结构

格式：

```text
[阶段代号]-[类型代号]-[序号]
```

阶段代号：

| 代号 | 含义 |
|------|------|
| BAS | 基本面与产品身份 |
| DLV | 交易与交付结构 |
| ESS | 产品本质分析 |
| RES | 结果路径与边界 |
| CMP | 竞争格局与替代方案 |
| USR | 用户画像与用户问题 |
| DEC | 用户决策链与成交阻力 |
| REP | 口碑与外部证据 |
| POS | 推广定位与话术 |
| CON | 内容资产与转化入口 |

类型代号：

| 代号 | 含义 |
|------|------|
| F | 事实 |
| A | 分析 |
| R | 建议 |
| W | 警告 |
| Q | 问题 |
| S | 话术 |

示例：
- `BAS-F-01`：基本面第 1 条事实。
- `DLV-A-01`：交易与交付结构第 1 条分析。
- `ESS-A-03`：产品本质分析第 3 条分析。
- `RES-W-01`：结果路径与边界第 1 条风险。
- `CMP-W-01`：竞争分析第 1 条风险。
- `POS-R-02`：推广定位第 2 条建议。
- `DEC-Q-01`：用户决策链中的第 1 个关键问题。
- `CON-R-01`：内容资产与转化入口第 1 条建议。
- `USR-Q-04`：用户问题库第 4 个问题。

## 置信度

| 等级 | 名称 | 定义 | 宣传使用 |
|------|------|------|----------|
| A | 高置信 | 官方资料、主管部门、合同文件、用户提供的一手资料可直接支持 | 可直接使用，仍需保留条件边界 |
| B | 较高置信 | 多个独立来源交叉支持，或事实链较完整 | 可谨慎使用，避免绝对化 |
| C | 中等置信 | 单一来源、资料不完整、逻辑推导合理但未验证 | 需人工确认后使用 |
| D | 低置信 | 缺少来源或只是初步猜测 | 只做内部判断，不用于宣传 |

## 来源链

事实类信息必须写：
- 来源类型：用户资料、官方公开信息、媒体、口碑、联网搜索、内部资料
- 来源细节：文件名、页码、链接、截图编号、用户输入位置
- 时效标注：年份、招生季、政策期、获取日期

分析类信息必须写：
- 推导依据：引用事实编号或用户问题编号
- 推导逻辑：一句话说明推导路径
- 不确定性：说明需要补什么证据

建议类信息必须写：
- 适用场景
- 依赖的事实或分析编号
- 风险边界

警告类信息必须写：
- 风险原因
- 触发条件
- 替代表述或处理方式

## 可宣传强度

| 强度 | 定义 | 使用方式 |
|------|------|----------|
| 强 | 有 A/B 级证据支撑，风险可控 | 可作为主卖点 |
| 中 | 有证据但边界较多 | 可作为辅助卖点 |
| 弱 | 证据不足或表达容易误导 | 只做内部参考或内容铺垫 |
| 不可宣传 | 涉及红线、隐私、过度承诺、来源弱 | 禁止用于外部内容 |

## 信息索引

每次输出完整输入包时，最后必须生成信息索引：

| 编号 | 内容摘要 | 类型 | 置信度 | 来源或依据 | 可否直接用于宣传 |
|------|----------|------|--------|------------|------------------|

索引必须覆盖正文中出现的所有编号。

## 编号使用规则

- 同一份输入包内编号不能重复。
- 事实编号只用于事实，不承载观点。
- 分析编号必须能追溯到事实编号。
- 建议编号必须能追溯到事实或分析编号。
- 风险编号必须进入推广红线或风险提醒。
- 新增信息追加编号，不重排旧编号。
- 修订版本中废弃的信息，标注“已废弃”，保留引用痕迹。

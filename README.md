# 项目洞察助手PRO

`项目洞察助手PRO` 是一个面向项目拆解、产品判断和推广交接的本地 skill。它只执行项目洞察第一阶段：把原始项目资料整理成一份干净、可读、可判断、可复用的项目洞察报告。

它适合教育咨询、课程培训、服务产品、项目招商、内容推广前的“先看清项目”场景。默认输出不给用户展示编号、置信度和证据账本，优先呈现业务结论、目标人群、交易交付、结果边界、成交阻力和日常推广打法。

## 能解决什么

- 看清产品到底是什么，用户付费后买到什么。
- 拆清交易结构、交付方式、费用边界和结果边界。
- 判断项目适合谁，不适合谁，谁付款，谁使用，谁影响决策。
- 找出成交阻力、反对理由和需要补证的材料。
- 给出日常推广应该先打的人群、内容资产和转化入口。
- 为后续选题、素材筛选、宣传写作、短视频脚本和咨询话术提供输入包。

## 默认输出结构

- 一页结论：产品是什么、核心价值、是否值得推广、优先人群、关键红线。
- 产品拆解：主体、交付物、交易结构、交付方式、费用边界。
- 结果路径：项目能支持到哪一步，哪些结果依赖用户条件、第三方规则或当年政策。
- 分析结论：价值内核、结构性优劣势、替代方案压力。
- 目标人群：适合人群、不适合人群、购买者、使用者、影响者、反对者。
- 成交阻力：真实反对理由、回应方向、需要补充的证据。
- 推广打法：主打人群、主打痛点、卖点顺序、内容角度、转化入口、话术边界。
- 内容资产：日常可生产的内容方向、适合平台、承接方式、销售使用方式。
- 下游写作指令：给选题、素材、文章、短视频、咨询话术使用。

## 适合场景

- 项目宣传前，需要先形成项目洞察报告。
- 拿到招生简章、项目手册、课程页、官网介绍、内部资料后，需要拆成可执行的推广输入。
- 产品卖点很多，但缺少清晰人群、交付边界和销售转化入口。
- 团队要把一个项目交给内容、投放、销售或咨询顾问继续使用。

## 不适合场景

- 完整项目洞察流水线。
- 选题框架生成。
- 素材筛选。
- 文章成稿。
- 普通资料摘要。

这些任务应交给后续流程或其他 skill。

## 安装方式

把 `skill/项目洞察助手PRO/` 整个目录复制到你的本地 skills 目录，例如：

```text
~/.codex/skills/项目洞察助手PRO/
```

复制完成后，确认目录中存在：

- `SKILL.md`
- `references/01-项目洞察PRO流程.md`
- `references/02-信息编号与置信度规则.md`
- `references/03-证据与推广红线规则.md`
- `templates/项目宣传写作输入包模板.md`
- `learnings.md`

## 使用方式

在支持本地 skill 的工具中输入类似指令：

```text
启动项目洞察PRO，分析下面这份项目资料，输出项目洞察报告。
```

如果需要审计版，可以明确说明：

```text
请输出审计版，包含编号、置信度和证据账本。
```

## 发布前校验

在仓库根目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/validate.ps1
```

校验内容包括：

- 必要文件是否齐全。
- skill 运行目录中是否误放 `README.md`。
- `SKILL.md` 中的内部名称是否为 `project-insight-pro`。
- skill 运行文件中是否包含个人本机路径。

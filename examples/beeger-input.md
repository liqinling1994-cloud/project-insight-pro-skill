# BeeGer 样例输入

> 用途：展示项目洞察助手PRO如何把公开产品资料拆成项目宣传写作输入包。  
> 来源：BeeGer 官网公开页面、官网页面内容梳理稿。  
> 官网链接：https://beeger.cn/

## 1. 项目基础信息

- 项目名称：BeeGer
- 一句话定位：你的 AI 运营官。
- 核心表达：懂业务，会干活。
- 核心任务：用 1 位 AI 运营官调度多个按流程分工的 AI 智能体，替用户完成图文、音频、视频的生产与发布。
- 当前官网入口：https://beeger.cn/

## 2. 官网公开信息摘录

BeeGer 面向内容运营场景，强调围绕产品、客户、账号定位、平台规则、素材方向和内容目标工作。

官网展示的工作链路包括：

1. 判断与选题：围绕热点、素材、卖点和目标客户进行方向判断。
2. 生产：生成图文、音频、视频脚本和 AI 合成视频，并匹配账号风格。
3. 发布与复盘：分发到微信公众号、小红书、抖音、B站、知乎等平台，并持续复盘优化。

官网强调的适用人群包括：

- 老板 / 创始人
- 专业顾问 / 行家
- 矩阵操盘手
- 代运营 / 内容服务商
- 一人公司

## 3. 官网梳理稿中的关键信息

首屏核心 slogan 已锁定：

- 懂业务，会干活。
- 无限复制最懂业务的人。

官网结构建议：

1. 首屏：3 秒说清 BeeGer 是 AI 运营官。
2. 适合谁：按角色讲具体问题。
3. BeeGer 怎么干活：说明从业务资料到内容执行的路径。
4. 素材怎么变内容：展示已有素材如何复用。
5. 案例 / 行业案例：预留案例卡片。
6. 预约演示：引导用户带真实业务资料跑一遍。

## 4. 角色与痛点

| 用户类型 | 主要卡点 | BeeGer 对应价值 |
|---|---|---|
| 企业老板 / 内容团队负责人 | 业务判断集中在少数核心成员，账号扩张后协同压力上升 | 放大公司里最懂业务的人的运营能力 |
| 专业顾问 / 高客单价服务者 | 专业判断很强，但持续内容表达消耗大 | 把专业判断翻译成客户看得懂、愿意读的内容 |
| 矩阵操盘手 / 代运营 / 内容服务商 | 账号多、客户多、素材多，人力管理越来越重 | 同一批素材按不同账号、平台和人群重写 |
| 一人公司 / 轻团队老板 | 自己懂业务，却被内容执行拖住 | 用系统调度内容队伍，承担重复执行活 |
| 培训机构负责人 / 课程服务商 | 咨询方案、课程内容、客户问答很多但分散 | 把分散内容整理成文章、案例、问答和社群素材 |

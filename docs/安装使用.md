# 多平台安装与使用教程

本文说明用户拿到 GitHub 项目地址后，如何在 Codex、Claude Code / CC、ZCode、WorkBuddy / Work Body 等支持 `SKILL.md` 的工具里快速安装项目洞察助手PRO。

## 1. 获取项目

项目地址：

```text
https://github.com/liqinling1994-cloud/project-insight-pro-skill
```

推荐用 Git 获取：

```powershell
git clone https://github.com/liqinling1994-cloud/project-insight-pro-skill.git
```

也可以在 GitHub 页面点击 `Code` -> `Download ZIP`，解压后进入项目目录。

真正需要安装的是这个目录：

```text
skill/项目洞察助手PRO/
```

目标目录中必须看到这些文件：

```text
项目洞察助手PRO/
├─ SKILL.md
├─ learnings.md
├─ references/
│  ├─ 01-项目洞察PRO流程.md
│  ├─ 02-信息编号与置信度规则.md
│  └─ 03-证据与推广红线规则.md
└─ templates/
   └─ 项目宣传写作输入包模板.md
```

运行目录内不要放 `README.md`、安装说明、发布说明或网页展示文件。这些文档留在仓库根目录和 `docs/` 中。

## 2. Codex 安装

把 `skill/项目洞察助手PRO/` 复制到 Codex 的 skills 目录。Windows 常见位置：

```text
%USERPROFILE%\.codex\skills\项目洞察助手PRO\
```

安装步骤：

1. 获取仓库。
2. 复制 `skill/项目洞察助手PRO/` 整个目录。
3. 粘贴到 Codex skills 目录。
4. 确认目标目录根层存在 `SKILL.md`。
5. 新开 Codex 会话，使用触发词启动。

测试指令：

```text
启动项目洞察PRO，基于下面的项目资料，输出一份项目洞察报告。
```

审计版调用：

```text
启动项目洞察PRO，输出审计版，包含编号、置信度、来源链和证据账本。
```

## 3. Claude Code / CC 安装

Claude Code / CC 的目录随用户配置变化。只要环境支持 skills，就按同一结构放置：

```text
skills/项目洞察助手PRO/
├─ SKILL.md
├─ references/
└─ templates/
```

安装要点：

- 使用 GitHub 项目中的 `skill/项目洞察助手PRO/` 作为源目录。
- 保持 `SKILL.md`、`references/`、`templates/` 的相对位置。
- 如果 Claude Code 支持项目级 skills，优先放进当前项目可读取的位置。
- 如果它只读取用户级 skills，就放到用户级目录。

常用触发词：

```text
项目洞察助手PRO
项目洞察PRO
项目洞察报告
项目宣传写作输入包
深度项目洞察
目标人群分析
日常推广打法
```

测试方式：

```text
项目洞察助手PRO，分析下面这份项目资料。先输出干净阅读版，不要展开审计附录。
```

如果没有触发，检查：

- skill 目录是否被 Claude Code 读取。
- `SKILL.md` 是否在目录根层。
- frontmatter 中 `name: project-insight-pro` 是否存在。

## 4. ZCode 安装

把 `skill/项目洞察助手PRO/` 复制到 ZCode 可读取的 skills 目录。常见结构：

```text
ZCodeProject/.Codex/skills/项目洞察助手PRO/
```

安装步骤：

1. 获取仓库。
2. 复制 `skill/项目洞察助手PRO/`。
3. 粘贴到 ZCode 的项目级或用户级 skills 目录。
4. 刷新 ZCode 的 skill 列表，必要时重启工作区。
5. 在会话中输入测试指令。

测试指令：

```text
启动项目洞察PRO，基于下面这段项目资料，输出一份干净阅读版项目洞察报告。
```

验收结果：

- 输出开头直接说明产品是什么。
- 输出包含适合人群、成交阻力、推广打法和下游写作指令。
- 默认输出不展示编号、置信度和证据账本。

## 5. WorkBuddy / Work Body 安装

WorkBuddy / Work Body 按通用 `SKILL.md` 环境处理。只要它能读取 `SKILL.md`，安装逻辑一致。

推荐结构：

```text
工作区/
└─ skills/
   └─ 项目洞察助手PRO/
      ├─ SKILL.md
      ├─ references/
      └─ templates/
```

触发方式：

```text
使用项目洞察助手PRO，基于下面资料输出项目宣传写作输入包。
```

验收方式：

- 先看是否按“产品是什么 -> 用户买到什么 -> 适合谁 -> 怎么推广”的顺序输出。
- 再看是否给出下游选题、文章、短视频和销售话术指令。
- 最后确认高风险表达是否被降级处理。

## 6. 案例演示样例

仓库提供 BeeGer 样例，方便测试安装后的输出效果。

输入样例：

```text
examples/beeger-input.md
```

输出样例：

```text
examples/beeger-output.md
```

你可以把 `beeger-input.md` 的内容作为测试资料，要求 skill 输出一份报告，再和 `beeger-output.md` 对照。

## 7. 发布前校验

在仓库根目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/validate.ps1
```

校验通过时会看到：

```text
Validation passed: project-insight-pro skill package is complete.
```

如果校验失败，优先检查：

- `skill/项目洞察助手PRO/` 是否存在。
- `SKILL.md` 是否包含 `name: project-insight-pro`。
- `references/` 是否至少有 3 个规则文件。
- `templates/` 是否至少有 1 个模板文件。
- 运行目录中是否误放 `README.md`。
- 运行目录文件中是否出现本机绝对路径。

## 8. 输出验收标准

一份合格输出应覆盖：

- 产品是什么。
- 用户实际购买的东西。
- 交易与交付结构。
- 价值判断和结果边界。
- 适合人群和不适合人群。
- 用户决策链。
- 成交阻力和回应方向。
- 日常推广人群。
- 内容资产和转化入口。
- 推广禁区和替代表述。
- 下游写作指令。

常见误用：

- 直接要求它写文章成稿。
- 直接要求它生成全年选题框架。
- 直接要求它筛选素材。
- 把未经核验的信息当作对外强卖点。

这些任务应在项目洞察报告完成后交给后续流程。

# 安装与使用

## 安装

把仓库中的 `skill/项目洞察助手PRO/` 目录复制到本地 skills 目录。

常见目标目录：

```text
~/.codex/skills/项目洞察助手PRO/
```

安装后确认存在这些文件：

```text
SKILL.md
learnings.md
references/01-项目洞察PRO流程.md
references/02-信息编号与置信度规则.md
references/03-证据与推广红线规则.md
templates/项目宣传写作输入包模板.md
```

## 推荐调用方式

默认报告：

```text
启动项目洞察PRO，基于下面的项目资料，输出一份项目洞察报告。
```

带审计附录：

```text
启动项目洞察PRO，输出审计版，包含编号、置信度、来源链和证据账本。
```

资料不足时：

```text
先基于现有资料做有限分析，再列出最小补证清单。
```

## 验收标准

一份合格输出应覆盖：

- 产品是什么。
- 用户实际购买的东西。
- 交易与交付结构。
- 价值判断和结果边界。
- 适合人群和不适合人群。
- 用户决策链。
- 成交阻力和回应方向。
- 日常推广人群。
- 内容资产和转化入口。
- 推广禁区和替代表述。
- 下游写作指令。

## 常见误用

- 直接要求它写文章成稿。
- 直接要求它生成全年选题框架。
- 直接要求它筛选素材。
- 把未经核验的信息当作对外强卖点。

这些任务应在项目洞察报告完成后交给后续流程。

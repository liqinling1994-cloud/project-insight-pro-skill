# 发布前校验

本文用于在发布或同步 GitHub 前检查项目洞察助手PRO的展示、案例、安装说明和运行目录。

## 1. 检查展示页

展示页文件：

```text
docs/index.html
```

需要确认：

- 首屏有项目洞察助手PRO的定位。
- 角色插图正常显示。
- 页面包含运行逻辑、输出效果、BeeGer 样例、安装教程和发布检查区。
- BeeGer 官网链接可以点击打开。

## 2. 检查 BeeGer 样例

输入样例：

```text
examples/beeger-input.md
```

输出样例：

```text
examples/beeger-output.md
```

需要确认：

- BeeGer 以真实产品样例出现，没有抢占项目洞察助手PRO的主定位。
- 样例输出展示了产品定义、用户买到什么、价值边界、适合人群、成交阻力、推广打法和下游指令。
- 样例中保留 BeeGer 官网链接。
- 高风险表达已经降级，没有承诺爆款、成交或增长。

## 3. 检查多平台教程

教程文件：

```text
docs/安装使用.md
```

需要确认：

- 已覆盖 Codex。
- 已覆盖 Claude Code / CC。
- 已覆盖 ZCode。
- 已覆盖 WorkBuddy / Work Body。
- 已说明如何通过 GitHub 项目地址获取仓库。
- 每个平台都有放置结构、触发指令和验收方式。

## 4. 检查展示图片资产

资产目录：

```text
assets/
```

需要确认：

- `hero-project-map.png`
- `workflow-pipeline.png`
- `beeger-case-dashboard.png`
- `output-handoff.png`

检查标准：

- 角色形象统一。
- 角色穿黑西装白衬衫。
- 每张图只有一个主要动作。
- 标签短、少、可读。
- 背景干净，有留白。
- 图片应为生成后的 PNG 成图，不使用手工 SVG 占位图。

## 5. 运行校验脚本

在仓库根目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/validate.ps1
```

通过时输出：

```text
Validation passed: project-insight-pro skill package is complete.
```

## 6. GitHub 同步前确认

准备同步前再检查：

- README 是否已经同步本次展示逻辑。
- `docs/index.html` 是否是最终版本。
- `examples/` 是否包含 BeeGer 样例。
- `scripts/validate.ps1` 是否通过。
- Git diff 中没有无关文件和本机隐私路径。

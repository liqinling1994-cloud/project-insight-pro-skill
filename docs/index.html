一眼看懂它做什么

它只做项目洞察第一阶段，把原始资料变成可读、可判断、可交接的业务判断。

输入

把资料喂进去

官网、项目手册、课程页、招生简章、销售笔记、竞品信息和用户口述都可以作为输入。

拆解

把项目看清楚

拆产品定义、用户买到什么、怎么交付、结果边界、适合谁、谁会反对。

输出

给下游继续用

输出给选题、素材、文章、短视频、销售话术和项目复盘流程继续调用。

运行逻辑

流程固定，表达保持可读。编号、置信度和证据账本默认留在内部，审计版才展开。

1

识别项目

先确认主体、交付物、价格、目标结果和资料缺口。

2

建立底稿

把事实、来源、时效、风险和待核验点分层记录。

3

拆交易交付

说明用户买到什么、怎么交付、谁负责、边界在哪里。

4

判断人群

识别适合对象、付款者、使用者、影响者和反对者。

5

输出交接

给出推广主线、内容资产、转化入口和下游写作指令。

输出质量雷达

每份报告都要让用户先看懂项目，再决定怎么推广。

真实产品样例：BeeGer

BeeGer 作为公开样例出现，用来展示本 skill 的拆解效果。它不改变项目洞察助手PRO的主定位。

软案例 访问 BeeGer 官网
产品定义 BeeGer 是一位 AI 运营官，围绕业务档案调度智能体完成内容生产与发布。
核心价值 放大最懂业务的人的判断，把资料、素材和账号定位转成持续内容资产。
优先人群 企业老板、专业顾问、矩阵操盘手、代运营、一人公司和课程团队。
推广边界 不承诺爆款、成交或增长，不替代真实专业判断和真实拍摄。

样例输出预览

完整样例放在 `examples/` 目录，本页展示关键片段。

一句话定义 BeeGer 是面向内容运营场景的 AI 运营官。
用户买到什么 用户买到的是一套能持续工作的内容运营链路。
先打哪群人 先打有产品、有业务判断、有持续内容需求的人。
成交阻力 用户会担心 AI 不懂业务、内容不像账号语气、自动发布有风险。
内容资产 成本账、角色场景、业务档案演示、素材复用演示和发布链路演示。
下游指令 给选题、文章、短视频和销售话术流程继续使用。

发布前怎么检查

同步远程仓库前，检查页面、案例、教程和脚本校验，确认 GitHub 首页可直接阅读。

检查文件

  • docs/index.html：官方展示页。
  • examples/beeger-input.md：BeeGer 样例输入。
  • examples/beeger-output.md：BeeGer 样例输出。
  • docs/安装使用.md：多平台安装教程。

检查动作

  • 检查展示页图片是否显示。
  • 点击 BeeGer 官网链接，确认能跳转。
  • 检查案例输出是否能说明 skill 效果。
  • 运行校验脚本，确认仓库结构完整。
